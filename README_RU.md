# 🤖 Big Dream Lab — AI платформа продаж и поддержки

AI-powered Telegram-бот, который автоматизирует квалификацию лидов, поддержку клиентов и планирование встреч для [Big Dream Lab](https://bigdreamlab.kz) — образовательной компании в Астане, Казахстан.

## Что делает

Один Telegram-бот обрабатывает всё:

- **Клиент отправляет заявку** → AI классифицирует лида (hot/warm/cold), генерирует персонализированный ответ, сохраняет в CRM и уведомляет менеджера
- **Клиент задаёт вопрос** → RAG-система ищет ответ в документах компании и отвечает со ссылкой на источник
- **Менеджер одобряет лида** → бот отправляет клиенту ответ и создаёт встречу в Google Calendar
- **Менеджер настраивает бота** → автоответ вкл/выкл, фильтры уведомлений, аналитика — всё через команды Telegram

## Функционал

### Квалификация лидов
- **AI-классификация** — Gemini классифицирует входящих лидов как hot, warm или cold на основе намерения покупки, упоминания компании, бюджета и сроков
- **Генерация персонализированного ответа** — AI пишет индивидуальный ответ каждому лиду с учётом его сообщения, компании и категории
- **Обогащение данных о компании** — локальная база крупных компаний Казахстана (Kaspi Bank, Halyk Bank, Kolesa Group, Air Astana и др.) с автоматическим fuzzy matching через LLM-нормализацию
- **Мультиязычность** — отвечает на том же языке, что и клиент (русский, английский)
- **Google Sheets CRM** — каждый лид записывается с таймстемпом, категорией, информацией о компании, сгенерированным ответом

### RAG база знаний
- **Ответы на основе документов** — отвечает на вопросы о курсах, ценах, грантах и информации о компании из проиндексированных документов
- **Семантический поиск** — векторная база ChromaDB с эмбеддингами sentence-transformers
- **Указание источников** — каждый ответ содержит ссылки на документы-источники
- **Определение намерения** — Gemini автоматически отличает запросы на покупку (лиды) от информационных вопросов (RAG)

### Инструменты менеджера
- **Автоответ ВКЛ/ВЫКЛ** — `/on` отправляет ответы автоматически, `/off` требует ручного одобрения через inline-кнопки
- **Human-in-the-loop** — когда автоответ выключен, менеджер просматривает AI-ответ и одобряет (✅) или отклоняет (❌)
- **Фильтр уведомлений** — `/filter_hot` показывает только горячих лидов, `/filter_hot_warm` для горячих+тёплых, `/filter_all` для всех
- **Аналитика** — `/analytics` показывает статистику лидов, топ курсы, топ компании за последние 2 дня и за всё время
- **Интеграция с Google Calendar** — после одобрения лида бот спрашивает клиента об удобном времени, парсит ответ и создаёт событие в Google Calendar

### Команды бота
| Команда | Описание |
|---------|----------|
| `/on` | Включить автоответ |
| `/off` | Выключить автоответ |
| `/status` | Текущие настройки |
| `/analytics` | Аналитика по лидам |
| `/filter_hot` | Уведомления только для горячих |
| `/filter_hot_warm` | Для горячих и тёплых |
| `/filter_all` | Все уведомления |

## Быстрый старт

### Требования
- Docker Desktop
- Аккаунт ngrok (бесплатный)
- Google Cloud проект с включёнными Sheets и Calendar API
- Telegram-бот (через @BotFather)
- Gemini API ключ (Google AI Studio)
- Groq API ключ (console.groq.com)

### Установка

1. Клонировать репозиторий:
```bash
git clone https://github.com/aibarC/sales-lead-qualifier.git
cd sales-lead-qualifier
```

2. Скопировать шаблон окружения и заполнить ключи:
```bash
cp .env.example .env
```

3. Запустить ngrok-туннель:
```bash
ngrok http 5678
```

4. Обновить `WEBHOOK_URL` в `.env` вашим ngrok URL.

5. Запустить все сервисы:
```bash
docker compose up -d --build
```

6. Открыть n8n на `http://localhost:5678`, импортировать `workflow.json`.

7. Настроить credentials в n8n (Telegram, Gemini, Google Sheets, Google Calendar).

8. Опубликовать workflow.

### Проверка

- n8n: `http://localhost:5678`
- RAG health: `http://localhost:5000/health`
- RAG тест: `http://localhost:5000/ask?q=Сколько стоит курс Unity?`

## Запланированные фичи

- **Follow-up напоминания** — автоматическое напоминание менеджеру, если клиент не ответил в течение 24 часов
- **Проверка свободных слотов** — проверка доступности менеджера в Calendar перед записью
- **Автоответ по расписанию** — `/schedule 09:00-18:00` для автоответа по времени
- **Казахский язык в RAG** — мультиязычные эмбеддинги (напр. multilingual-e5-large) для полной поддержки казахского
- **Загрузка PDF через Telegram** — менеджер отправляет PDF боту → автоматическая индексация в RAG

## Дисклеймер

Это **демо-прототип**, созданный как портфолио-проект. В production-среде необходимы следующие доработки:

- **Безопасность**: управление секретами (напр. HashiCorp Vault), HTTPS без ngrok, rate limiting
- **Масштабируемость**: n8n cloud или self-hosted с PostgreSQL, горизонтальное масштабирование RAG-сервиса
- **Надёжность**: обработка ошибок, retry-логика, логирование и мониторинг (напр. Grafana)
- **Данные**: production база данных вместо Google Sheets, интеграция с CRM (HubSpot, Salesforce)
- **Мультиязычный RAG**: мультиязычные эмбеддинги (напр. multilingual-e5-large) для полной поддержки казахского языка
- **Тестирование**: unit-тесты, интеграционные тесты, нагрузочное тестирование
